═══════════════════════════════════════════════════════════════
REVIEW CHECKLIST (apply to every code submission)
═══════════════════════════════════════════════════════════════

Memory Safety:
  □ All pointers initialized before use
  □ No buffer overflows (array bounds checking)
  □ malloc/free paired correctly (no leaks, no double-free)
  □ Stack usage within limits (no deep recursion on MCU)

Hardware Safety:
  □ Volatile qualifier on all hardware registers and ISR-shared variables
  □ No floating-point in ISRs (unless FPU is explicitly configured)
  □ Proper register read-modify-write sequences (|= and &=~)
  □ Peripheral clocks enabled before register access
  □ Pin configurations match the target board's datasheet

Concurrency:
  □ Critical sections protect shared resources
  □ No blocking delays inside ISRs
  □ Proper use of atomic operations or mutexes
  □ Volatile on all ISR-shared variables

Logic:
  □ State machines have default/error states
  □ All switch cases have break statements (or fallthrough is intentional)
  □ Loop termination conditions are correct
  □ Edge cases handled (zero values, max values, overflow)

Communication Protocols:
  □ I2C addresses are 7-bit (not accidentally 8-bit shifted)
  □ SPI clock polarity and phase match the slave device
  □ UART baud rate matches on both ends
  □ Proper timeout handling on all blocking communication calls
